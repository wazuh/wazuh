# Issue #34915 - Evaluate and Implement FIM IT Parallelization on Linux Workflows

## Scope
- `.github/workflows/5_testintegration_fim-tier-0-1-lin.yml`
- `.github/workflows/5_testintegration_fim-tier-2-lin.yml`
- `.github/workflows/5_builderpackage_agent-linux.yml`

## Changes implemented
- Parallelized Tier 0-1 workflow by splitting the combined `build` job into a 2-shard matrix:
  - `tier-0` → `pytest --tier 0`
  - `tier-1` → `pytest --tier 1`
- Unlike the Windows workflow, Linux has a single combined job that covers build, install and
  test. Each shard therefore runs the full build+install+test cycle independently.

> **Note on Linux architecture:** The Linux FIM workflow has no separate build/run-test job
> split (unlike Windows). The matrix is applied to the single `build` job, so both shards
> rebuild the agent. A future optimization opportunity is to extract the build stage into a
> separate job and share the artifact across shards.

## Artifact reuse from builder workflow
### Evaluation
- Feasibility: Pending validation on real runs after merge queue execution.
- Required coupling (run-id/commit lookup, retention, permissions): High for cross-workflow
  artifact handoff.
- Measured gain: Pending. Unlike Windows, the Linux combined job includes compilation; if the
  build artifact from `5_builderpackage_agent-linux.yml` could be reused, both shards would
  skip rebuilding — a potentially significant gain.
- Risk/complexity impact: Medium-High if implemented across independent workflows.

## Final recommendation
- Tier 0-1 parallelization: Implemented in workflow definition, pending runtime validation.
- Tier 2 parallelization: Pending decision with additional measurements (current sample of 2 runs is too sparse).
- Builder artifact reuse: Higher potential impact than on Windows (avoids full recompilation per shard); pending decision.
- Trade-offs: Expected lower wall-clock time in tier 0-1 at the cost of higher concurrent runner usage and double build overhead per workflow run (until artifact reuse is resolved).
