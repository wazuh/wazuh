# Release notes

## Highlights

- New detection Engine replacing `analysisd`, with a YAML-based ruleset, KVDB lookups, and event enrichment (IOC matching, GeoIP, filters). ([#33493](https://github.com/wazuh/wazuh/issues/33493)) ([#34477](https://github.com/wazuh/wazuh/issues/34477))
- Native event forwarding to the Wazuh Indexer through the `indexer-connector` module; Filebeat is no longer required. ([#33124](https://github.com/wazuh/wazuh/pull/33124))
- Cluster-by-default deployment model: every Wazuh Server installation runs as a cluster node, removing the distinction between clustered and non-clustered deployments. ([#31295](https://github.com/wazuh/wazuh/issues/31295)) ([#27434](https://github.com/wazuh/wazuh/issues/27434))
- Complete Agent/Manager separation: the Agent and Manager are now fully independent products with distinct installation paths, system users (`wazuh` vs. `wazuh-manager`), configuration files, daemon names, and runtime artifacts. Both can be installed and run simultaneously on the same host without conflicts, and all coupling through the reserved agent ID `000` has been removed. ([#33378](https://github.com/wazuh/wazuh/issues/33378))
- New stateless event model: agents send lean, self-contained events while the Manager centrally enriches them with agent identity, group membership, and cluster context at the `remoted` boundary, guaranteeing a consistent metadata schema for every event reaching the Indexer and removing `wazuh-db` from the ingestion hot path. ([#33269](https://github.com/wazuh/wazuh/issues/33269))
- New state synchronization model: agents own and persist their module state (FIM, System Inventory, SCA) locally and reconcile it with the Manager through a dedicated synchronization protocol with deterministic versioning and checksum-based recovery, replacing the legacy `rsync` path and reducing network traffic and server-side processing. ([#29533](https://github.com/wazuh/wazuh/issues/29533)) ([#31838](https://github.com/wazuh/wazuh/issues/31838))
- Vulnerability Detection now uses the Wazuh Indexer as the authoritative CVE data source, fed by the CTI service, with CVSS v4.0 support. ([#34919](https://github.com/wazuh/wazuh/issues/34919)) ([#35623](https://github.com/wazuh/wazuh/issues/35623))
- Revamped Role-Based Access Control (RBAC) management, including an upgrade mechanism for existing RBAC configurations. ([#27706](https://github.com/wazuh/wazuh/issues/27706))
- Engine metrics collection, normalization, and indexing pipeline for internal observability. ([#35470](https://github.com/wazuh/wazuh/issues/35470))
- Configurable queue watermarks across manager components (Inventory Sync, `indexer-connector`, Engine, Vulnerability Detector, and `remoted`), with graceful overflow handling and saturation warnings, bounding memory use and reducing OOM risk under load. ([#35579](https://github.com/wazuh/wazuh/issues/35579))

## Breaking changes

- No in-place upgrade from 4.x: Wazuh Manager 5.x packages refuse to install on hosts with an existing 4.x (or earlier) installation. A fresh install plus the documented migration process is required. ([#36317](https://github.com/wazuh/wazuh/issues/36317))
- As part of Agent/Manager separation, the installation path moved from `/var/ossec` to `/var/wazuh-manager`; `ossec.conf` â†’ `wazuh-manager.conf`, the XML root `<ossec_config>` â†’ `<wazuh_config>`, `internal_options.conf` â†’ `wazuh-manager-internal-options.conf`, `ossec.log` â†’ `wazuh-manager.log`, the system user/group `wazuh` â†’ `wazuh-manager`, and the control binary and daemons were prefixed accordingly (`wazuh-control` â†’ `wazuh-manager-control`, `wazuh-analysisd` â†’ `wazuh-manager-analysisd`, and so on; the `wazuh-manager.service` systemd unit name is unchanged). Any automation or tooling referencing these paths, filenames, daemon names, or the system user must be updated. ([#33378](https://github.com/wazuh/wazuh/issues/33378)) ([#35479](https://github.com/wazuh/wazuh/issues/35479))
- Filebeat was removed as the log-shipping component; event forwarding configuration moves to the `indexer-connector` block. ([#33124](https://github.com/wazuh/wazuh/pull/33124))
- The cluster is always enabled; the `cluster.disabled` configuration option was removed. ([#31295](https://github.com/wazuh/wazuh/issues/31295)) ([#27434](https://github.com/wazuh/wazuh/issues/27434))
- Deprecated manager daemons were removed: `ossec-authd`, `wazuh-agentlessd`, `wazuh-maild`, `wazuh-dbd` ([#30922](https://github.com/wazuh/wazuh/issues/30922)), and `wazuh-csyslogd`, `wazuh-integratord`, `wazuh-reportd` are no longer shipped ([#30919](https://github.com/wazuh/wazuh/issues/30919)).
- Deprecated CLI tools `manage_agents` and `agent-auth` were removed. ([#30924](https://github.com/wazuh/wazuh/issues/30924))
- Syslog remote input (`remoted` `syslog` connection mode, port 514) was removed. ([#34729](https://github.com/wazuh/wazuh/issues/34729))
- Ruleset migration: the Engine replaces `analysisd` and its ruleset format â€” XML decoders are replaced by YAML decoders ([#29585](https://github.com/wazuh/wazuh/issues/29585)), and CDB lists are replaced by KVDB key-value databases ([#31053](https://github.com/wazuh/wazuh/issues/31053)).
- The Vulnerability Detector offline feed was removed. CVE data is now fetched exclusively from the CTI service, which requires the Manager to register and maintain outbound connectivity to it. Air-gapped or unregistered deployments will have no vulnerability data. The registration state (`Pending`, `Polling`, `Denied`, `Available`) is queryable via the Dashboard and Server API. ([#34919](https://github.com/wazuh/wazuh/issues/34919)) ([#31906](https://github.com/wazuh/wazuh/issues/31906))
- Agent metadata fields in Vulnerability Detection events moved from `host.*` to `agent.*` in the Indexer; index templates were updated accordingly. Custom dashboards, queries, and integrations targeting the previous field paths must be updated. ([#30535](https://github.com/wazuh/wazuh/issues/30535))
- Threat-related fields were aligned under the `wazuh` namespace in the Engine enrichment; queries and dashboards referencing the previous threat field locations must be updated. ([#35885](https://github.com/wazuh/wazuh/issues/35885))
- The VirusTotal integration was removed along with `integratord`. Real-time VirusTotal lookups have no direct replacement; IOC-based enrichment is now handled inline by the Engine against the Wazuh IOC database. Existing `integratord`-based VirusTotal configurations no longer apply. ([#36326](https://github.com/wazuh/wazuh/issues/36326))
- Active Response was redesigned around the Wazuh Indexer Alerting/Notifications plugins (new `active_response` channel type): the manager `ar.conf` file and the `<active-response>` and `<command>` configuration blocks were removed ([#34663](https://github.com/wazuh/wazuh/issues/34663)), and the `PUT /active-response` API endpoint was removed ([#34783](https://github.com/wazuh/wazuh/issues/34783)). ([#34606](https://github.com/wazuh/wazuh/issues/34606))
- The server-side OpenSCAP module was removed; use SCA instead. ([#31028](https://github.com/wazuh/wazuh/issues/31028))
- Inventory-related API endpoints were removed ([#31299](https://github.com/wazuh/wazuh/issues/31299)), along with the legacy API security configuration endpoints ([#28425](https://github.com/wazuh/wazuh/issues/28425)).
- Legacy configuration surfaces, database schemas, and compatibility layers were removed in the server cleanup. ([#34608](https://github.com/wazuh/wazuh/issues/34608))
- Manager index names changed to align with the new CTI and engine index aliases. ([#35525](https://github.com/wazuh/wazuh/issues/35525))
- Rootcheck server-side simplification: the manager-side rootcheck database, synchronization path, and read-only API were removed; findings are now indexed through the standard alert pipeline. ([#31478](https://github.com/wazuh/wazuh/issues/31478))
